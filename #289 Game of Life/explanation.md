The brute force solution is to use a second mxn matrix and go cell by cell, perform the logic, and then come to a conclusion if it should be alive or dead and set that value in the new matrix. This is O(nxm) time and O(nxm) space complexity due to checking each cell and having a duplicate matrix of the same size as the original.

A better, but not the best, solution would be to use a smaller data structure to store which cells need to be changed to alive or dead. We could use a map that's keys is live or dead and the value can be a list of tuples representing the cell's coordinates. After walking through the matrix once to fill the hashmap, i would iterate through the live and dead lists and change the cells accordingly. This is O(nxm) time and O(n+m) space complexity, which is better, but we can make it O(1) constant time complexity

To get O(1) space complexity, i need to create temporary states by assigning a specific transorming state to a specific int greater or less than 1 or 0. Lets say a cell that is alive and is dying is -1 while a cell that is dead and is coming back to life is 2. This way, we will do one pass checking each cell and changing it to a transormative value if necessary. on the second pass, we will change the cells that contain a transformative value according to what it is. This way, time complexity is O(mxn) and space is O(1).
